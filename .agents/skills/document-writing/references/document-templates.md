# Document Templates

Use these as lightweight skeletons. Adapt them to the repository context instead of filling every heading by force.

## README

1. What the product is
2. Who it is for
3. Core capabilities
4. Technical or operational constraints
5. Installation or quick start
6. Links to deeper docs

## Requirements Note

1. Background and problem
2. Target users
3. Goals and non-goals
4. Core requirements
5. Constraints and assumptions
6. Open questions

## Design Memo

1. Problem statement
2. Proposed approach
3. Key tradeoffs
4. Data or responsibility boundaries
5. Risks
6. Follow-up work

## Operating Guide

1. Purpose
2. Preconditions
3. Procedure
4. Failure cases and recovery
5. Expected outcome
