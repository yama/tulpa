# Tulpa

> エンジニア・エージェント・クライアント、3者の思念をAIで繋ぐOSS

---

## なぜTulpaか

クライアントは「こういう人に来てほしい」という像を持っている。エンジニアは「こんなことができる・こんな仕事がしたい」という思念を持っている。担当者は「このエンジニアとこの案件は合うはずだ」という勘を持っている。

それなのに、マッチングは手作業のテキストで、データはそれぞれ別の画面に散らばったまま。

Tulpaは**案件・クライアントを中心にデータを繋げ、AIが3者の思念を形にする橋渡し**となることを目指します。

---

## 主な機能

**エンジニア向け**

- 毎月のExcel報告書をなくす。スマホから記録して、そのまま提出
- 契約・支払通知・スキルシートが一か所に。「いくら・いつ振り込まれるか」が一目でわかる
- 年間カレンダーで稼働ペースと休暇余裕を把握

**エージェント向け**

- エンジニアCRM・クライアントCRM（営業活動・次アクション管理）
- Claude APIによるスキルシート×案件要件のAIマッチング
- 候補者サマリーの自動生成

**クライアント向け**

- 稼働報告のオンライン確認・差戻し

## 技術スタック

レンタルサーバーでも動作する軽量設計です。

| カテゴリ | 技術 |
|--------|------|
| バックエンド | PHP 8.2+ / Laravel 12 |
| フロントエンド | Alpine.js + htmx（ビルド不要）/ Tailwind CSS |
| データベース | MySQL 8.0+ / MariaDB 10.6+ |
| オプション | Claude API（AIマッチング）/ Slack API（通知連携）|
| モバイル | PWA（スマホのホーム画面から起動可能）|

## インストール

```bash
git clone https://github.com/yama/tulpa.git
cd tulpa
composer install --no-dev
cp .env.example .env
php artisan key:generate
php artisan migrate
```

詳細は [インストールガイド](docs/installation.md) を参照してください。
